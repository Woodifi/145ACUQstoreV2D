1. Introduction

QStore IMS (Inventory Management System) is a self-contained browser-based application for managing Australian Army Cadet Q-Store operations. It runs entirely in the browser using local storage — no internet connection or server is required for day-to-day use.

Key capabilities: Inventory tracking with five-state condition breakdown, photo support, and QR codes · Loan issue and return workflow with printable vouchers and overdue tracking · Cadet self-service equipment request workflow (AB189-style) with QM approval · Nominal roll (cadets and staff) with company / platoon / section grouping · Stocktake with full condition breakdown · AAC QStore order PDF import with IMS receive workflow · Tamper-evident audit log · Optional cloud backup via Microsoft OneDrive · Role-based access for OC, QM, Staff, Cadets, and Read-Only users.

Data storage: All data is stored in IndexedDB in the browser on the device running the app. Data does not leave the device unless cloud sync is configured or a manual backup is exported.

2. Getting started

First login

  1. Open qstore.html in a modern browser (Chrome, Edge, Firefox, or Safari)
  2. Select Administrator from the user list
  3. Enter PIN: 0000
  4. You will be prompted to change the default PIN immediately — do this before proceeding

Initial setup

  1. Navigate to Settings
  2. In Unit Details, enter unit name, unit code, state, OC/QM name, email, and rank
  3. Upload a unit logo if desired
  4. Click Save unit details

Adding users

  1. Go to Users (OC only, visible in the navigation bar)
  2. Click + Add User and fill in name, username, role, service number, and initial PIN
  3. After saving, the PIN is displayed once — note it down and give it to the user verbally
Only the OC can manage user accounts and PINs. Users cannot change their own PINs. If a user forgets their PIN, the OC must reset it via Users → Reset PIN.

4. Inventory

Condition breakdown

Every inventory item tracks a five-state breakdown. Svc is automatically calculated — only enter the non-serviceable quantities.

UnserviceableU/SYesDamaged or non-functional
In RepairReprYesTemporarily unavailable (being repaired)
Calibration DueCalYesMust be calibrated before issue
Written OffW/OYesBeyond repair — pending Board of Survey

Adding an item OC / QM

  1. Click + Add item
  2. Enter NSN (4-2-3-4 format), name, category, authorised qty, on hand qty
  3. Enter U/S, Repr, Cal, W/O as needed — Svc is calculated automatically
  4. Enter a location — the location field is searchable
  5. Click Save item

Printing

  • ⎙ Print stock — PDF of currently-visible items with condition breakdown
  • ⎙ QR codes — printable QR label sheet
  • ⎙ AB174 — Board of Survey form for written-off items

5. Loans

Issuing equipment OC / QM

  1. Go to Loans → Issue tab (or press Shift+N)
  2. Select the borrower (cadets and staff are included in the borrower list)
  3. Select the purpose, set issue date and due date
  4. Add item lines — search by name or NSN, set quantity
  5. Click Issue

A loan reference (LN-XXXX) is assigned automatically.

Loan purposes

Initial Issue · Annual Camp · Training Activity · Parade Night · Field Exercise · Ceremonial · Course Attendance · Other

Initial Issue: Due date set to 6 years from today, locked. Never flagged overdue. Marks permanent personal issue for the duration of enlistment.

Returning equipment OC / QM

  1. Go to Loans → Return tab (or press Shift+R)
  2. Find the loan and click Return
  3. Set return condition (Serviceable / Unserviceable / Write-off)
  4. Click Confirm return

All Loans tab

Filter by Active / Returned / Overdue / All. Discharged borrowers' loans are flagged with a red Discharged badge and have been automatically recalled.

6. Equipment requests (self-service)

Submitting a request (all roles)

  1. Go to Requests → Submit Request
  2. Select purpose, required-by date, and add item lines
  3. Click Submit Request

A reference (REQ-XXXX) is assigned. You can also Import AB189 PDF to pre-fill from a digitally-filled form.

Request statuses

QM/OC review OC / QM

From the Pending tab: Approve & Issue — creates loan records immediately; Approve — marks approved without loans; Deny — enter a denial reason.

7. Cadets / nominal roll

Adding a person OC / QM

  1. Click + Add cadet/staff
  2. Enter service number, surname, given names, rank, and platoon details
  3. Click Save

Person type (cadet or staff) is derived automatically from the rank entered.

Unit sub-structure

Configure Company → Platoon → Section hierarchy in Settings → Unit sub-structure. Once set, the add/edit form shows cascading dropdowns and the nominal roll displays group-band headers.

Cadet discharge — automatic loan recall

When a cadet is deactivated (Active unticked and saved), the system immediately sets the due date on all active loans to today and flags those loans Discharged. A cadet_discharge audit entry lists all recalled loan references. The QM must process the physical return through the normal return workflow.

Kit checklist PDFs

Individual: Click Equipment on any cadet row → ⎙ Print kit checklist. Batch: Click ⎙ Batch checklists in the toolbar — generates one PDF per person with active loans in the current filtered view.

8. Stocktake

Running a stocktake OC / QM

  1. Go to Stocktake and enter counts for each item's five condition columns
  2. Total sums all five columns; Variance compares to system on-hand quantity
  3. Use the category filter to work section by section — drafts save automatically
  4. Click Finalise stocktake and review the breakdown before confirming
Warning: Finalising is irreversible. Written-off items must be formally struck off charge via Board of Survey (AB174).

Blank stocktake worksheet PDF

Click ⎙ Worksheet to generate a blank printed count sheet for use on the floor before entering data into the system.

9. AAC QStore orders

Importing an order PDF OC / QM

  1. Download the PDF from the AAC QStore website
  2. Go to Orders → Import PDF Order
  3. Select the PDF — order number, category, date, requestor, and all line items are extracted automatically
  4. Review the editable fields and click Save Order

Receiving into IMS OC / QM

  1. Open the order and click Approve & Receive into IMS
  2. Adjust quantities if needed (set a line to 0 to skip it)
  3. Click Confirm & Update IMS
Warning: Receiving is irreversible from the Orders page. Correct affected items manually in Inventory if an error is made.

10. Audit log

The Audit page provides a tamper-evident log of all actions taken in the system. Available to OC and QM. Each entry shows: sequence number, timestamp, action type, user, and description.

Click Verify chain to confirm the cryptographic chain is intact — a broken chain indicates potential tampering. Export currently-filtered entries via ⬇ Export CSV or ⬇ Export JSON.

11. Settings

Settings is accessible to the OC only.

Unit details

Enter unit name, unit code, state, OC/QM name and email, QM rank. Upload a unit logo. This information appears in the app header, on the login screen, and on generated documents.

Unit sub-structure

Configure Company → Platoon → Section hierarchy. Add companies, platoons, and sections using the tree editor. Click Save structure when done.

Categories

Manage item categories used in Inventory and Stocktake. Drag to reorder, type to add, click ✕ to remove. Click Save categories to apply or Reset to defaults to restore.

Security — auto-lock

SettingOptionsDefault
Auto-lock after idle5 / 10 / 15 / 30 min / 1 hour15 minutes (enforced from first login)

Auto-lock cannot be disabled — the minimum is 5 minutes. The lock also fires immediately on wake from sleep or when the browser tab becomes visible again if the idle period has elapsed during that time. Changes take effect immediately without a reload.

12. Cloud sync (OneDrive)

Cloud sync backs up data to Microsoft OneDrive automatically. It is optional — the app works fully offline without it.

Setup OC

  1. Go to Settings → Cloud sync
  2. Note the Redirect URI — register it in Azure exactly as shown
  3. Enter your Azure Application (Client) ID
  4. Click Save then Sign in
Cloud sync requires HTTPS. It is unavailable when opened as a file:// URL.

13. Backup and restore

Exporting a backup OC

  1. Go to Settings → Data backup & restore
  2. Click Export backup
  3. Choose whether to password-protect the file:
    • With password — encrypted .qstore file (AES-256-GCM, PBKDF2 310,000 iterations) — recommended
    • Without password — plain .json file
  4. Store the file off-device in a secure location
Warning: Backup files contain names, service numbers, and equipment records — treat as sensitive personnel documents. There is no password recovery for encrypted backups. Always keep at least 3 recent backups, at least one off-device.

Importing a backup OC

  1. Click Import backup
  2. Type OVERWRITE to confirm — this replaces all current data
  3. Select the .json or .qstore file
  4. If encrypted, enter the password when prompted

14. PIN security

PIN rules and management

PINs must be exactly 4 digits. PINs are managed exclusively by the OC — no user can view or change their own PIN. After any PIN is set or reset, it is displayed once for the OC to note down. It cannot be retrieved again.

Auto-lock (idle timeout)

Configured in Settings → Security → Auto-lock after idle (5 / 10 / 15 / 30 min / 1 hour — minimum 5 minutes; cannot be disabled). The default of 15 minutes is active from first login even before the setting is explicitly saved.

The lock triggers on inactivity and immediately on wake from sleep or when the browser tab becomes visible again if the idle period elapsed during that time. Entering the correct PIN resumes the session. Failed unlock attempts follow the same escalating lockout as login.

15. Two-factor authentication (2FA)

2FA adds a second sign-in step: after your PIN you enter a 6-digit code from an authenticator app. Works fully offline. Compatible with Google Authenticator, Microsoft Authenticator, Authy, 1Password, Bitwarden, and any RFC 6238-compliant app.

Recommendation: OC and QM accounts should have 2FA enabled. The Users page shows an amber warning badge for privileged accounts without 2FA.

Setting up 2FA (per user)

  1. Log in, then go to Settings → Two-factor authentication
  2. Click Set up two-factor authentication
  3. Step 1 — scan the QR code with your authenticator app, or enter the secret key manually
  4. Step 2 — enter the 6-digit code from your app to verify it is working
  5. Step 3 — save the 8 single-use backup codes (print or store in a password manager); tick the confirmation checkbox; click Enable two-factor authentication

Signing in with 2FA

After entering your PIN, enter the 6-digit code from your app — it auto-submits on the sixth digit. If rejected, check your device clock is accurate (TOTP codes are time-sensitive).

Backup codes

On the TOTP code screen, click Use a backup code instead and enter the 8-character code. Each code is single-use. View remaining count in Settings. Regenerate via Manage 2FA → Regenerate backup codes.

Warning: If you exhaust all backup codes and lose your authenticator app, you cannot log in. Regenerate backup codes before they run low. Contact your support provider if all access is lost.

Disabling 2FA

Settings → Two-factor authentication → Manage 2FA → Disable two-factor authentication. Confirm — the TOTP secret and backup codes are permanently removed.

16. OC PIN recovery

The OC account has an additional recovery mechanism. Set it up before it is needed.

Generating a recovery code OC

  1. Go to Settings → OC PIN recovery
  2. Click Generate new code
  3. The 12-character recovery code is displayed once — write it down immediately
  4. Store it off-device: printed copy in the unit safe

Using a recovery code

  1. On the login screen, select the OC account
  2. Click Forgot PIN?
  3. Enter the 12-character recovery code
  4. Enter and confirm a new PIN
  5. Generate a new recovery code immediately from Settings
Warning: Recovery codes are one-use only. Without a recovery code, a lost OC PIN cannot be recovered. Contact admin@seanscales.com.au for assistance.

17. Issue kits

Issue kits are pre-defined item bundles that pre-fill the loan issue form with a single click — useful for Initial Issue or camp packing lists.

Creating a kit OC / QM

  1. Go to Inventory → ⊞ Kits → + New kit
  2. Name the kit, add item lines with quantities
  3. Click Create kit

Using a kit

  1. Go to Loans → Issue tab and click ⊞ Load kit
  2. Select the kit — item lines are pre-filled
  3. Adjust quantities as needed, then proceed with the issue

18. QR codes

Printing labels

On the Inventory page, click ⎙ QR codes to generate a PDF with one label per item (respects active search/category filters), including NSN, name, and QR code.

Scanning labels

Click ⌖ Scan on the Inventory page. Allow camera access if prompted. Point at a QR code label — the matching item is highlighted in the inventory list.

19. Troubleshooting

App won't load / blank screen
Ensure JavaScript is enabled. Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac). Check browser console (F12) for errors.
User forgot their PIN
The OC must click Reset PIN on the Users page and give the new PIN to the user verbally.
Forgot OC PIN — have recovery code
On the login screen select the OC account and click Forgot PIN?. Enter the recovery code and set a new PIN. Generate a new recovery code immediately afterward.
Forgot OC PIN — no recovery code
There is no bypass without a recovery code. Contact admin@seanscales.com.au for assistance. Always generate and store a recovery code before it is needed.
Stock levels look wrong
Run a stocktake to reconcile physical counts. Check the Audit log for unexpected adjustments.
Cloud sync not working
Ensure the redirect URI in Azure exactly matches the URL shown in Settings. Azure app must have Files.ReadWrite, offline_access, openid, and profile permissions. Try signing out and back in. Cloud sync requires HTTPS or localhost.
Audit chain verification fails
Data has been modified outside the application. Import from your most recent known-good backup.
Data missing after browser update
IndexedDB is tied to the browser profile and origin. If you switch browsers or reinstall, data may not be present. Always keep current backup exports off-device.
Session locked after waking from sleep
This is the intended security behaviour — the idle timer accounts for time elapsed while the machine was asleep. Enter your PIN to resume.
